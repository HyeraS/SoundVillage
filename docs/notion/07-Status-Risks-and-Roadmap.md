# 현재 상태, 리스크, 로드맵

## 현재 상태 요약

프로토타입의 핵심 사용자 흐름과 Supabase 연결은 구현되어 있다. 프로덕션 빌드는 성공한다. 다만 실제 오디오 파일, DB 스키마 마이그레이션, 자동 테스트가 저장소에 없고 ESLint 오류가 남아 있어 재현 가능한 운영 상태로 보기에는 보완이 필요하다.

## 확인된 ESLint 오류

2026-06-05 `npm run lint` 기준 10건:

| 영역 | 내용 |
|---|---|
| `app/page.js` | Effect 안에서 상태 갱신 함수를 직접 호출 |
| `AnnotationPanel.js` | 렌더 중 `Math.random()` 호출, 렌더 중 ref 접근 |
| `WorldMap.js` | 렌더 중 `Date.now()` 호출 |
| `ZoneMap.js` | 렌더 중 `Date.now()` 호출, 렌더 중 ref 접근 |

## 우선순위별 리스크

### P0: 데이터와 실행 전제

| 리스크 | 영향 | 권장 조치 |
|---|---|---|
| 실제 오디오 파일 없음 | 핵심 기능 재생 불가 | `public/audio` 또는 CDN 자산 준비 |
| DB 스키마/RPC 정의 없음 | 새 환경 재현 불가 | Supabase migration SQL 추가 |
| RLS 정책 미확인 | 공개 anon key로 데이터 오용 가능 | 읽기/쓰기 정책과 제약 검토 |

### P1: 데이터 무결성

| 리스크 | 영향 | 권장 조치 |
|---|---|---|
| 투표 insert와 count 증가가 비원자적 | 집계 불일치 | 단일 RPC/트랜잭션으로 통합 |
| RPC 오류 미검사 | 조용한 데이터 유실 | 각 RPC 결과 확인 및 실패 처리 |
| 반복 투표 방지 없음 | 투표 왜곡 | DB unique 제약과 정책 추가 |
| 스킵/투표 오류 숨김 | 사용자는 성공으로 인식 | 재시도 및 오류 UI 추가 |

### P1: 연구 설계

| 리스크 | 영향 | 권장 조치 |
|---|---|---|
| 개인 진행도와 전역 진행도 혼재 | 참여자 오해 가능 | 라벨 분리 또는 개인 진행도 저장 |
| 패널 닫기만 해도 수집 처리 | 미제출 소리를 재시도 못함 | 닫기와 스킵 의미 분리 |
| 후보가 인기순으로만 표시 | 선호 편향 강화 | 무작위화/균형 샘플링 검토 |
| participant/session 검증 없음 | 데이터 정리 비용 증가 | 실험 배정 시스템 또는 형식 검증 |

### P2: 코드 품질과 유지보수

| 리스크 | 영향 | 권장 조치 |
|---|---|---|
| 대형 단일 컴포넌트 | 수정 위험 증가 | 렌더러/데이터/루프 분리 |
| Zone 정의 중복 | 색상/라벨 불일치 가능 | 단일 Zone config로 통합 |
| 미사용 `VillageScene` | 혼란과 유지 비용 | 제거 또는 명시적 대체 UI로 채택 |
| 상위 폴더 중복 파일 | 잘못된 파일 수정 위험 | 비교 후 보관/삭제 결정 |
| 자동 테스트 없음 | 회귀 탐지 어려움 | 데이터 유틸과 핵심 흐름 테스트 추가 |

## 권장 로드맵

### 1단계: 재현 가능한 환경

- Supabase migration과 RLS를 저장소에 추가
- 오디오 자산 배치 방식 확정
- `.env.example` 추가
- README를 실제 프로젝트 내용으로 교체

### 2단계: 데이터 안전성

- 투표 저장을 트랜잭션 RPC로 통합
- 참여자/세션 및 중복 투표 제약 추가
- 모든 저장 실패를 사용자에게 알리고 재시도 지원
- 개인 진행도와 전역 집계를 분리

### 3단계: 품질

- ESLint 오류 해결
- 핵심 유틸 단위 테스트
- Stage 1/2 통합 테스트
- 모바일, Safari, 저속 네트워크 검증

### 4단계: 확장

- 사운드 카탈로그 관리 도구
- 운영/분석 대시보드
- 후보 표현 샘플링 전략
- 접근성 개선 및 키보드 전용 흐름

## 의사결정이 필요한 항목

- 패널 닫기를 스킵으로 볼 것인가, 미처리로 볼 것인가
- Zone 진행도를 개인 기준으로 보여줄 것인가, 전체 수집 현황으로 보여줄 것인가
- Stage 2에서 투표가 없어도 청취 시간을 저장할 것인가
- 한 참여자가 같은 sound를 여러 세션에서 반복 처리할 수 있는가
- 오디오를 Git/Vercel 정적 자산으로 둘 것인가, 외부 스토리지/CDN을 사용할 것인가
