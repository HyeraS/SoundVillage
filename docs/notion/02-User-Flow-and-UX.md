# 사용자 경험과 화면 흐름

## 전체 상태 흐름

```text
start
  -> world
  -> zone
  -> annotate / Stage 1
  -> annotate / Stage 2
  -> zone + 완료 토스트
  -> world
```

화면 상태는 `app/page.js`의 `screen` 값으로 관리한다. 가능한 값은 `start`, `world`, `zone`, `annotate`다.

## 1. 시작 화면

컴포넌트: `StartPanel`

1. 참여자 ID와 세션 ID를 입력한다.
2. 두 값이 모두 공백이 아닐 때 시작 버튼이 활성화된다.
3. 세션 ID 필드에서 Enter를 눌러도 시작할 수 있다.
4. 값은 별도 검증 없이 문자열로 메모리에 저장된다.

## 2. 월드맵

컴포넌트: `WorldMap`

- 캐릭터는 중앙에서 시작한다.
- 방향키 또는 WASD로 움직인다.
- 모바일에서는 좌하단 D-Pad를 사용한다.
- 포털 근처에서 Enter 또는 Space를 누르면 Zone에 진입한다.
- HUD는 Supabase 전체 기록 수와 Zone별 진행도를 표시한다.

포털 배치:

| 위치 | Zone |
|---|---|
| 좌상 | Forest |
| 우상 | Water |
| 중앙 | City |
| 좌하 | Music |
| 우하 | Mystery |

## 3. Zone 맵

컴포넌트: `ZoneMap`

- Zone별 배경색, 장식, 소리 아이템 심볼이 다르다.
- 최대 8개의 소리 아이템이 맵의 사전 정의 위치에 생성된다.
- 캐릭터와 아이템이 충돌하면 0.5초 수집 효과 후 AnnotationPanel이 열린다.
- ESC 또는 HUD의 월드맵 버튼으로 나갈 수 있다.
- 모든 아이템이 수집되면 완료 모달이 표시된다.

주의: 수집은 제출뿐 아니라 패널 닫기와 건너뛰기에도 적용된다. 현재 세션에서 같은 아이템이 다시 나타나지 않게 하기 위한 동작이다.

## 4. Stage 1: 표현 입력

컴포넌트: `AnnotationPanel` 내부 `Stage1Panel`

1. 재생 버튼으로 소리를 듣는다.
2. 최대 80자의 자유 표현을 입력한다.
3. 자신감 `Low`, `Medium`, `High` 중 하나를 고른다. 기본값은 `Medium`이다.
4. 제출하면 `annotations` 테이블에 저장한다.
5. 저장 성공 후 Stage 2로 이동한다.

자신감 매핑:

| UI 값 | 저장 값 |
|---|---|
| Low | 1 |
| Medium | 3 |
| High | 5 |

건너뛰기를 누르면 빈 표현, `is_skipped=true`, `skip_reason=user_skip`으로 저장을 시도하고 패널을 닫는다.

## 5. Stage 2: 공감 투표

컴포넌트: `AnnotationPanel` 내부 `Stage2Panel`

1. 같은 `sound_id`에 대한 다른 표현을 최대 6개 불러온다.
2. 자신의 표현과 동일한 텍스트는 제외한다.
3. 후보는 누적 투표 수가 높은 순서로 표시된다.
4. 여러 후보를 선택해 투표할 수 있다.
5. 선택이 없으면 `공감 없음`으로 종료할 수 있다.

투표 저장이 실패해도 화면은 완료 처리된다. 이 동작은 사용자 흐름을 막지 않지만 데이터 유실이 사용자에게 표시되지 않는다는 의미다.

## 6. 완료와 복귀

- 제출/투표 흐름이 끝나면 해당 `sound_id`가 현재 세션의 `collectedIds`에 추가된다.
- ZoneMap으로 돌아오고 약 2.2초 동안 완료 토스트가 표시된다.
- 전체/Zone 집계가 Supabase에서 다시 조회된다.

## 입력 및 접근성

| 입력 | 동작 |
|---|---|
| 방향키/WASD | 캐릭터 이동 |
| Enter/Space | 월드맵에서 Zone 진입 |
| Escape | Zone에서 월드맵 복귀 |
| 화면 D-Pad | 모바일 이동 |
| Annotation 배경/닫기 버튼 | 패널 닫기 및 해당 소리 수집 처리 |

현재 접근성 제약:

- 게임 캔버스가 SVG와 시각적 충돌에 크게 의존한다.
- 일부 버튼에는 명시적 `aria-label`이 없다.
- 포커스 이동과 스크린리더용 게임 설명이 제한적이다.
- 전역 `overflow: hidden` 및 스크롤바 숨김이 적용된다.
